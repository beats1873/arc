Economy

Arcturus has a full coin economy. Members earn coins from multiple sources — daily/weekly claims, game wins, emote reactions, birthday bonuses, and leveling up. Coins can be spent in the shop. Daily earning from automated sources can be capped per-source and overall from the Economy dashboard section.

Ways to Earn Coins

SourceAmountHow
/daily+10Once every 24 hours
/weekly+50Once every 7 days
/birthday claim+1–100 (random)Once per year, within one day of your birthday
Trivia winConfigurable (default +5)First correct answer to a trivia question
Hangman win+lives remainingSolving hangman — coins equal the lives remaining at win
Emote+1Each emote reaction command used
Level up+10Each time you reach a new chat XP level

Daily Coin Caps

Configurable from Economy in the dashboard. All caps reset at midnight UTC. Each has an independent on/off toggle — disabling a cap does not clear its configured value.

CapApplies toDefault value
Overall Daily CapAll automated sources combined100 (off)
Hangman CapHangman wins only50 (off)
Trivia CapTrivia wins only50 (off)

When a user hits their cap, the usual coin award message is replaced with a notification that their daily limit has been reached. If a cap partially limits a payout (e.g. only 3 coins remain before the cap), the reduced amount is still awarded and noted in the message.

⚡ XP & Leveling

Chat and voice XP are tracked independently. Each has its own level curve, rank position, and role rewards.

How XP Works

  • Chat XP — A random amount between the configured Min and Max XP is awarded per message, subject to a per-user cooldown. Default: 15–25 XP, 60-second cooldown.
  • Voice XP — Awarded at a fixed rate per minute spent in a non-AFK voice channel. Default: 10 XP/min.
  • Ignore Channels — A single unified list blocks XP in both text and voice channels. Add a text channel to stop chat XP there; add a voice channel to stop voice XP there.
  • All XP gain can be disabled with the master toggle in the dashboard.

Rank Card — /rank [user]

A generated image showing the member's avatar, username, coin balance, and two separate rank badges:

  • Chat Rank (primary colour) — position by chat XP among all server members
  • Voice Rank (green) — position by voice XP among all server members

Below the badges, chat and voice XP progress bars each show current XP / required XP with the percentage displayed inline.

Leaderboard — /leaderboard

A generated image with three sections (top 5 each):

  • Coins — full-width section at the top, sorted by coin balance (gold)
  • Chat XP — top 5 by chat level with relative progress bars (primary colour)
  • Voice XP — top 5 by voice level with relative progress bars (green)

Chat XP and Voice XP sections sit side by side beneath the Coins section.

XP Roles

Map Discord roles to chat or voice level milestones separately in the dashboard. Roles are assigned when the threshold is crossed and are cumulative — lower-tier roles are always kept. Use Import Roles to backfill XP for members who already hold level roles.

🎂 Birthday

Members can register their birthday with the bot. At midnight in the configured server timezone, the bot posts a birthday embed in the designated channel for every member whose birthday it is that day.

Member Slash Commands

CommandDescription
/birthday setRegister your birthday (month and day). Stored privately — only the bot sees the date.
/birthday removeDelete your stored birthday from the bot.
/birthday claimClaim a special birthday coin bonus. Can only be claimed once per year, within one day of your actual birthday.

Dashboard Settings

  • Enabled toggle — Turns birthday announcements on or off.
  • Announcement Channel ID — The channel where birthday embeds are posted.
  • Timezone — Controls what "midnight" means for birthday checks. Use IANA timezone names such as America/New_York or Europe/London. Defaults to UTC.
  • Embed Title / Footer — Custom text shown in the birthday embed. Leave blank for the bot default.

🛒 Shop

The shop lets members spend their coins on items defined by admins. Items can optionally grant a Discord role on purchase and can be quantity-limited. All item fields — including the name — can be edited after creation directly from the dashboard.

Member Slash Commands

CommandDescription
/shop listBrowse all available items with prices, quantities, and descriptions.
/shop buy <item>Purchase an item by name. Deducts the coin cost and, if configured, grants the linked role immediately.

Dashboard — Managing Items

  • Name — Unique identifier used in /shop buy. Spaces are allowed.
  • Cost — Price in coins.
  • Quantity — Leave blank for unlimited stock. Once stock reaches 0 the item is unavailable for purchase.
  • Role — Optional. The Discord role to grant on purchase. Leave blank for items fulfilled manually.
  • Description — Shown in /shop list to explain what the item does.

Click the pencil icon next to any item to pre-fill the form with its current values. Change any field (including the name) and click Save Changes. Click Cancel to discard and return to add mode.

Manual Fulfilment

If an item has no role attached, the bot pings a configured Notification Role in a Notification Channel when someone buys it. Set both in the Fulfillment Notifications card at the top of the Shop section.

❓ Trivia

Trivia posts auto-scheduled questions in a configured channel. Members answer by typing in that channel. The first correct answer earns coins. Sessions can run continuously or be limited to a set number of questions. Answer checking is strict — answers must match exactly or have at most one typo (two for long words). Similar-sounding words (e.g. "Nigeria" for "Algeria") will not be accepted.

Slash Commands

CommandDescription
/trivia startStart a trivia session manually. Requires Manage Server or a Trivia Mod role.
/trivia stopStop the active trivia session. Requires Manage Server or a Trivia Mod role.

Dashboard Settings

  • Trivia Channel ID — Where questions are posted and answers are read.
  • Questions per Session — How many questions are asked before the session ends automatically. Set to 0 for unlimited.
  • Interval — Time between questions (hours + minutes). E.g. 0 hours 30 minutes = one question every 30 minutes.
  • Trivia Mod Roles — Roles allowed to start/stop trivia sessions via slash commands (in addition to Manage Server).
  • Session Control — Start or stop the live session directly from the dashboard.

Question Pool

Questions are stored in data/questions.json and managed from the Trivia Questions dashboard section. You can add, edit, search, and remove questions from the pool at any time. Changes take effect immediately.

🎮 Hangman

A prefix-based hangman game. A random word is chosen (from an API with a built-in categorised fallback word list) and members type letters or the whole word to guess. The word category (e.g. Animals, Places, Food) is announced at the start to help players. The winner receives coins equal to remaining lives.

Prefix Commands

CommandDescription
Start a new hangman game. The bot announces the word category before the game begins. Any member can start a game.

Dashboard Settings

  • Enabled toggle — Turns hangman on or off server-wide.
  • Restrict to Channel ID — If set, only works in that channel. Leave blank to allow any channel.

🎉 Giveaways

Run timed reaction-based giveaways. Members react with the configured emoji to enter. A winner is randomly selected when the timer expires. The giveaway embed shows only the prize and end time — no host attribution. Both the entry embed and the winner announcement embed are fully customisable from the dashboard.

Slash Commands

/giveaway end <id>Manage Server / Giveaway RoleEnd a giveaway early and pick a winner immediately.
/giveaway listManage Server / Giveaway RoleView all currently active giveaways with entry counts and end times.
/giveaway reroll <id>Manage Server / Giveaway RolePick a new winner from the existing entry pool for an ended giveaway.

Entry Embed Settings

  • Enabled toggle — Must be on before any /giveaway commands work.
  • Giveaway Roles — Additional Discord roles that can manage giveaways (in addition to Manage Server).
  • Entry Emoji — The reaction emoji members click to enter. Default: 🎉
  • Embed Colour — Accent colour of the giveaway post embed.
  • Embed Header / Body — Customise the giveaway post. Supports {prize} and {emoji}. Leave blank for bot defaults.

Winner Announcement Embed

A separate embed is posted when the giveaway ends. Customise it from the Winner Announcement Embed card in the Giveaways dashboard section.

  • Title — Heading of the winner announcement. Supports {prize}, {winner}, {entries}.
  • Body — Description text. Same variables available. Leave blank to show only the prize/winner/entries fields.
  • Embed Colour — Default: green.

Auto Role

Automatically grants a configured role to every member the moment they join the server. Useful for setting a default "Member" role, granting access to channels, or applying initial permissions without any manual work.

How it works

  • When enabled and a role is selected, the bot immediately adds the role to any new member on join.
  • The bot's own role must be ranked above the auto role in Server Settings → Roles, otherwise Discord will block the assignment.
  • The feature is independent of welcome messages — both can be enabled at the same time.

Dashboard Settings

  • Enabled toggle — Turn the feature on or off without clearing the configured role.
  • Role to Grant — The role assigned to every new member on join. Select from all roles in the server.

Webhook Builder

The Webhook Builder lets you compose rich Discord embeds and send them to any text channel — no external webhook URL needed. The bot creates and manages webhooks on your behalf automatically.

How it works

  • Select a Target Channel from the dropdown. The bot finds or creates a webhook in that channel when you send.
  • Optionally set a custom Display Name and Avatar to override the bot's identity for that message. You can upload an image directly — no external URL required.
  • Add a plain-text Message Content (appears above the embed) and one or more Embeds with full field support.
  • The Live Preview panel on the right renders your message exactly as it will appear in Discord, including Discord markdown formatting.
  • Click Save to store the message draft, then Send to post it. Use Update Message to edit an already-posted message in place.

Discord Markdown Reference

The description and field values support Discord's markdown syntax. A full reference is available at the Discord Markdown Guide. Common formatting:

  • **bold** — bold text
  • *italic* — italic text
  • __underline__ — underlined text
  • ~~strikethrough~~ — struck-through text
  • `inline code` — monospace code span
  • ```code block``` — multi-line code block
  • > quote — block quote
  • :emoji_name: — emoji shortcode (e.g. :heart: → ❤️)

🔑 Prefixes & Permissions

Command Types

  • Slash commands (/command) — Registered with Discord. Appear in the command picker. Available to all members unless the command requires Manage Server.
  • Prefix commands (+command by default) — Text-based commands triggered by the configured command prefix. Used for hangman and birthday info.
  • Emote commands (=emote by default) — Text-based commands triggered by the configured emote prefix. Used for all emote reactions.

Permission Levels

  • Everyone — /coins, /daily, /weekly, /rank, /leaderboard, /shop list, /shop buy, /birthday set/remove/claim, +hangman, all emote commands.
  • Trivia Mod (configured role) — /trivia start, /trivia stop.
  • Manage Server — All admin commands plus full dashboard access.

Dashboard Access

The dashboard is accessible only to members with Manage Server or Administrator permission in the server. Authentication is handled via Discord OAuth2 — log in with your Discord account and the bot verifies your permissions automatically.
